# Настройка реальной отправки Email

## Шаг 1: Получите App Password от Gmail

### Вариант A: Если у вас Gmail

#### 1. Включите двухэтапную аутентификацию

1. Откройте https://myaccount.google.com/
2. В меню слева выберите **"Безопасность"** (Security)
3. Найдите **"Двухэтапная аутентификация"** (2-Step Verification)
4. Нажмите **"Начать"** и следуйте инструкциям
5. Введите номер телефона и подтвердите его

#### 2. Создайте пароль приложения (App Password)

1. После включения 2FA вернитесь на https://myaccount.google.com/
2. **"Безопасность"** → **"Пароли приложений"** (App Passwords)
   - Прямая ссылка: https://myaccount.google.com/apppasswords
3. Вас попросят ввести пароль - введите пароль от вашего Google аккаунта
4. В поле "Выберите приложение" выберите **"Почта"** (Mail)
5. В поле "Выберите устройство" выберите **"Другое"** (Other)
6. Введите название: **"ntFAST"**
7. Нажмите **"Создать"** (Generate)
8. **ВАЖНО:** Google покажет 16-значный пароль, например:
   ```
   abcd efgh ijkl mnop
   ```
9. **СКОПИРУЙТЕ ЕГО СЕЙЧАС!** Он больше не появится

#### 3. Подготовьте пароль

Удалите все пробелы из пароля:
```
abcd efgh ijkl mnop  →  abcdefghijklmnop
```

## Шаг 2: Настройте .env файл

Откройте файл `backend/.env` в текстовом редакторе (Notepad, VS Code, и т.д.)

### Для Gmail:

```env
# Email Configuration (SMTP)
SMTP_SERVER=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=ваш-email@gmail.com
SMTP_PASSWORD=abcdefghijklmnop
USE_REAL_EMAIL=true
```

**Замените:**
- `ваш-email@gmail.com` → ваш реальный Gmail адрес
- `abcdefghijklmnop` → App Password без пробелов (16 символов)
- `USE_REAL_EMAIL=true` → **обязательно true!**

### Для Yandex:

```env
SMTP_SERVER=smtp.yandex.ru
SMTP_PORT=587
SMTP_USERNAME=ваш-email@yandex.ru
SMTP_PASSWORD=ваш-пароль-от-yandex
USE_REAL_EMAIL=true
```

### Для Mail.ru:

```env
SMTP_SERVER=smtp.mail.ru
SMTP_PORT=587
SMTP_USERNAME=ваш-email@mail.ru
SMTP_PASSWORD=ваш-пароль-от-mailru
USE_REAL_EMAIL=true
```

## Шаг 4: Перезапустите backend сервер

### Остановите текущий сервер:

1. Найдите окно терминала где запущен backend
2. Нажмите **Ctrl+C**
3. Дождитесь полной остановки

### Запустите сервер заново:

```bash
cd backend
python -m uvicorn app.main:app --reload
```

Вы должны увидеть:
```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

## Возможные проблемы и решения

### Проблема: "Username and Password not accepted"

**Причины:**
1. Используете обычный пароль вместо App Password
2. Не включена двухэтапная аутентификация в Gmail
3. В пароле остались пробелы

**Решение:**
- Для Gmail: используйте только App Password (16 символов без пробелов)
- Убедитесь что 2FA включена
- Проверьте что пароль скопирован правильно

### Проблема: "Connection refused"

**Причины:**
1. Неправильный SMTP сервер или порт
2. Firewall блокирует соединение
3. Антивирус блокирует Python

**Решение:**
- Проверьте SMTP_SERVER и SMTP_PORT
- Отключите firewall временно для теста
- Добавьте Python в исключения антивируса

### Проблема: Email не приходит

**Причины:**
1. Письмо в папке Спам
2. USE_REAL_EMAIL=false (всё ещё demo mode)
3. Неправильный email адрес получателя

**Решение:**
- **Обязательно проверьте папку Спам!**
- Убедитесь что `USE_REAL_EMAIL=true`
- Проверьте email адрес на опечатки

### Проблема: Всё ещё показывает DEMO MODE

**Причины:**
1. Файл .env не сохранён
2. Сервер не перезапущен
3. .env находится не в папке backend

**Решение:**
- Сохраните .env (Ctrl+S)
- Полностью остановите и перезапустите сервер (Ctrl+C → uvicorn снова)
- Убедитесь что .env находится в `backend/.env`

## Безопасность

### НИКОГДА не публикуйте .env файл!

Файл `.env` содержит ваши пароли и уже добавлен в `.gitignore`.

Проверьте:
```bash
git status
```

Файл `.env` НЕ должен отображаться в списке файлов для commit.

### Не делитесь App Password

App Password даёт полный доступ к отправке email от вашего имени.
**Никому не показывайте этот пароль!**
